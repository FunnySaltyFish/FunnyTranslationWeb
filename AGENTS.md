# AGENTS.md

## 运行环境

- 当前工作 shell 是 Git Bash。优先使用 Bash 命令和路径写法。
- 使用 Playwright 预览或检查时，只使用 Chromium。不要使用 WebKit，因为当前环境没有安装。
- 这是静态 HTML 项目。除非明确要求，不要引入构建工具或复杂工程化依赖。

## 设计方向

- `design.md` 是最高优先级的风格参考。必须保留现有 ElevenLabs-like 方向：白色背景、暖灰分区、大留白、细衬线标题、轻边框和极轻阴影。
- 除非明确要求，不要重做视觉系统。不要随意修改全局背景、色调、间距节奏或整体气质。
- 页面目标气质是高级、克制、编辑感。避免素材堆叠、高饱和广告图、紫色/赛博渐变和通用 AI 味 UI。

## 素材使用

- 产品截图和视频位于 `assets/showcase/`。由 GPT-Image-2 生成的位于 `assets/generated`
- 不要为了“尽量用上素材”而把大量截图堆成图库。图片应该出现在对应功能说明旁边，用来解释具体功能。
- 产品视觉必须符合真实 App。不要生成或插入虚构 UI，避免造成虚假宣传。
- 生成首屏图时，应基于真实产品截图或克制文字排版，并匹配页面白色、暖灰、黑字的视觉语言。

## 媒体规则

- 保持图片和视频原比例。产品媒体不要使用 `object-fit: cover`。
- 优先用静态截图作为即时预览。
- 视频应按需加载：先展示图片预览，用户点击后再加载/播放视频。
- 首屏图片要保持适中尺寸，不能挤占主文案空间。

## 外部资源

- 避免运行时依赖海外 CDN，例如 Google Fonts。
- 优先使用本地资源和系统字体。
- 下载入口优先使用文档中已有的官方 API 地址。

## 交互要求

- 保持日夜间模式切换可用。
- 保持中英文切换可用。
- 保持移动端导航菜单可用。
- 进行明显 UI 修改后，要用 Playwright Chromium 检查桌面端和移动端。
- 移动端必须检查是否存在横向溢出。

## Git 与文件安全

- 不要回退用户的无关修改。
- `Transtation_官网信息整理.md` 可能有用户修改；除非明确要求，不要触碰。
- 完成前清理临时预览文件和测试产物。
