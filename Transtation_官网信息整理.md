## 2. 产品定位

译站不是单一翻译引擎的壳，而是一个把多翻译源、AI 模型、多模态输入和高频翻译工作流整合到一起的工具。

适合的人群包括：

- 学生
- 开发者
- 独立译者
- 跨境内容团队
- 内容创作者
- 需要频繁处理文本、图片、语音、漫画、网页和截图翻译的人

官网表达时，建议始终强调：

- 一次输入，多结果对照
- 基础能力零门槛
- 高级 AI 能力按需扩展
- 既适合新手，也适合专业用户
- 开源、可查、可信

## 3. 核心卖点

- 适合：
  - 查词

## 4. 细分功能亮点

### 4.1 图片翻译大模型

- 采用多模态模型驱动
- 能理解复杂图片上下文
- 可处理：
  - 手写内容
  - 竖排古文
  - 复杂表格
  - 混合版式

官网适合强调：

- 不只是 OCR
- 不只是抽字
- 还原版式与内容关系

### 4.2 语音翻译大模型

- 流式语音翻译
- 支持系统内录
- 支持扬声器输入
- 支持补充额外上下文
- 语音识别 + 大模型翻译 + 实时纠错

官网适合强调：

- 更适合课堂、会议、视频、播客
- 不只是转写
- 能持续纠错与补足上下文

### 4.3 漫画翻译大模型

- 一键“翻译 + 上色”
- 支持对图片内容进行自然语言编辑
- 适合强调：
  - 保留原始视觉结构
  - 减少违和感
  - 降低后期处理成本

### 4.4 声音朗读大模型

- 精选优质语音朗读引擎
- 相比普通机械朗读更有情感
- 支持：
  - 多语言混读
  - 情绪调控
  - 语速调控

### 4.5 长文翻译大模型

- 智能关联上下文
- 可识别并产出专业名词术语表
- 支持一次翻译长篇内容

官网适合强调：

- 连贯
- 可控
- 术语稳定
- 适合“整本级”内容

### 4.6 本地运行大模型翻译

- 支持本地大模型翻译
- 可用于：
  - 离线场景
  - 隐私场景
  - 网络不稳定场景

官网适合强调：

- 可离线
- 可控
- 更适合敏感内容

## 9. 官网文案必须遵守的原则

### 9.1 面向用户

官网中的每一句话都必须是写给最终用户看的。

绝对禁止出现：

- 写给开发者或站长自己的说明
- “把你给我的素材放进页面”
- “这部分是为了 SEO”
- “上方 / 下方我这样组织”
- “这部分补充结构化内容”
- 任何类似“解释网页是如何被制作出来”的句子

### 9.2 遵循《影响力》的原则

官网文案应潜移默化地使用以下原则：

- 互惠
  - 先强调免费基础能力
  - 先让用户获得价值，不先要求付费
- 承诺与一致
  - CTA 应鼓励用户先下载、先体验、先试一次
- 社会认同
  - 用用户数、下载量、专业评价、开源 Star 增强信任
- 喜好
  - 文案要专业但不装腔，清楚、克制、让人愿意继续读
- 权威
  - 强调多模型、多场景能力、开源透明、真实功能深度
- 稀缺
  - 如果未来有首发权益、限时折扣、测试资格，可以加
  - 当前未明确时不要乱写
- 共同体 / Unity
  - 让用户感到“这是为频繁处理翻译任务的人做的”

### 9.3 专业但好懂

整体风格要求：

- 专业
- 克制
- 不浮夸
- 对小白友好
- 对专业用户有说服力

### 9.4 不要写成“参数表”

官网不是文档页，不能只是堆功能点。

更有效的写法应该是：

- 先讲结果
- 再讲场景
- 再讲能力
- 最后讲价格和下载
